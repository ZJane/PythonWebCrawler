Java泛型

2018年08月07日 21:09:59
阅读数：2

在JDK1.5之后，在类的定义时，可以使用一个标记，此标记就表示类中属性或方法参数的类型，在使用的时候才动态地设置类型。即

泛型：

//TestDemo1.java
class Point{   //T即Type,是一个类型
    private Object x;   //此属性类型不明确，由Point类使用时动态决定
    private Object y;   //此属性类型不明确，由Point类使用时动态决定
    public void setX(){
        this.x = x;
    }
    public void setY(){
        this.y = y;
    }
    public Object getX(){
        return x;
    }
    public Object getY(){
        return y;
    }
}
public TestDemo1{
    public static void main(String[] args){
        Point p = new Point();
        p.setX(12);   //此时在运行程序之前，编译器不会自动排查是否会出错。但是执行之后会发生ClassCastException异常，因为Integer不能转换为String，如果直接加instanceof关键字就可能因为坐标种类的多样而导致很多坐标种类而被筛选掉
        p.setY("string");
        String x = (String) p.getX();
        String y = (String) p.getY();
        System.out.println("x="+x+",y="+y);

}

//TestDemo2.java使用泛型修改TestDemo1.java
class Point<T>{   //T即Type,只是一个标记，也可以是B、A等，但是在使用的时候需要为其设置具体的类型    
    private T x;   //此属性类型不明确，由Point类使用时动态决定
    private T y;   //此属性类型不明确，由Point类使用时动态决定
    public void setX(){
        this.x = x;
    }
    public void setY(){
        this.y = y;
    }
    public T getX(){
        return x;
    }
    public T getY(){
        return y;
    }
}
public TestDemo2{
    public static void main(String[] args){
        Point<Integer> p = new Point<Integer>();  //设置T为String（还有可以为包装类）
        //JDK1.7之后：
        //Point<Integer> p = new Point<>();
        p.setX(12);   
        p.setY("string");    //此时会自动排查参数错误
        //由于已经是Integer类型，所以就不用进行向下转型了，解决了类转换异常的问题
        System.out.println("x="+x+",y="+y);     

}

通配符：

先来看以下代码：

class Message<T>{
    private T msg;
    public void setMsg(T msg){
       this.msg = msg;
    }
    public T getMsg(){
        return msg;    
    }
}
public class TestDemo{
    public static void main(String[] args){
        Message<String> m = new Message<String>();
        m.setMsg("Hello Java");
        fun(m);  //引用传递
    }
    public static void fun(Message<String> temp){
        System.out.println(temp.getMsg());
    }
}

但是如果接收的参数是其他类型，那么就需要“牵一发而动全身”，需要同时修改fun方法、new参数类型。也就是说fun方法不能适应所有情况。

解决方法一：

class Message<T>{
    private T msg;
    public void setMsg(T msg){
       this.msg = msg;
    }
    public T getMsg(){
        return msg;    
    }
}
public class TestDemo{
    public static void main(String[] args){
        Message<String> m1 = new Message<String>();
        Message<Integer> m2 = new Message<Integer>();
        m1.setMsg("Hello Java");
        m2.setMsg(100);
        fun(m1);  //引用传递
        fun(m2);  //引用传递
    }
    public static void fun(Message temp){ //如果不设置泛型，就是Object，即相当于参数为Message<Object> temp
        System.out.println(temp.getMsg());
    }
}

但是此时fun方法会出现警告信息,因为只要不设置泛型，就会出现警告信息。

可能存在的问题是：

class Message<T>{
    private T msg;
    public void setMsg(T msg){
       this.msg = msg;
    }
    public T getMsg(){
        return msg;    
    }
}
public class TestDemo{
    public static void main(String[] args){
        Message<String> m1 = new Message<String>();
        Message<Integer> m2 = new Message<Integer>();
        m1.setMsg("Hello Java");
        fun(m1);  //引用传递
    }
    public static void fun(Message temp){ //如果不设置泛型，就是Object，即相当于参数为Message<Object> temp

        //"偷天换日"
        setMsg(100);
        System.out.println(temp.getMsg());  //此时输出的将是100，而不是所期望的Hello Java了
    }
}

所以现在应该解决的是：需要有一种方式使得fun()方法 接收一个类的任意的泛型，但是不能够进行参数的修改，只能进行参数内容的读取，这就需要采用通配符"?"来表示任意泛型了：

class Message<T>{
    private T msg;
    public void setMsg(T msg){
       this.msg = msg;
    }
    public T getMsg(){
        return msg;    
    }
}
public class TestDemo{
    public static void main(String[] args){
        Message<String> m1 = new Message<String>();
        Message<Integer> m2 = new Message<Integer>();
        m1.setMsg("Hello Java");
        fun(m1);  //引用传递
    }
    public static void fun(Message<?> temp){ //设置任意泛型为参数
        System.out.println(temp.getMsg());  
    }
}

在"?"通配符的基础上还有两个子通配符：

? extends：设置泛型上限，可以在方法声明上和方法参数上使用；

如：? extends Number：意味着泛型上限为Number类，可以设置Number或者是Number的子类作为泛型

? super：设置泛型下限，可以在方法参数上使用。

如：? super String：意味着只能够设置为或者是它的父类Object作为泛型。

泛型接口（泛型在接口上定义）：

//定义泛型接口时，取名有以下习惯：
//如果是接口则在前面加上字母I，
//如果是抽象类则在前面加上Abstract,
//如果是普通类则直接编写
interface IMessage<T>{  //设置泛型接口
    public void print(T t);
}

在接口上必须要定义其相应的子类，所以来讲如果要定义的子类有两种形式。

形式一：在子类继续设置泛型。

interface IMessage<T>{  //设置泛型接口
    public void print(T t);
}
class MessageImpl implements IMessage<T>{ //子类也继续使用泛型，并且父接口使用和子类同样的泛型标记
    public void print(T t){
        System.out.println(t);
    }
}
public class TestDemo{
    public static void main(String[] args){
        IMessage<String> msg = new MeaasgeImpl<String>();
        msg.print("Hello Java");
    }
}

形式二：在子类不设置泛型，而为父接口明确定义一个泛型类型。

interface IMessage<T>{  //设置泛型接口
    public void print(T t);
}
class MessageImpl implements IMessage<String>{ //子类不使用泛型，父接口使用泛型标记
    public void print(String t){
        System.out.println(t);
    }
}
public class TestDemo{
    public static void main(String[] args){
        IMessage<String> msg = new MeaasgeImpl();
        msg.print("Hello Java");
    }
}

泛型方法：

泛型方法不一定非要定义在支持泛型的类里面。而在之前所编写的所有存在有泛型的方法都是在泛型支持类里面定义的。

定义：

public class TestDemo{
    public static void main(String[] args){
        String str = fun("Hello Java");
        System.out.println(str.length());
    }
    public static <T> T fun(T t){ //T的类型由传入的参数类型决定
        return t;
    }
}

总结：

泛型解决的是向下转型所带来的安全隐患，其核心组成就是在声明类或接口的时候，不设置参数或属性的类型。

"?"表示的是接收任意的泛型类型，只能够取出，不能被修改
